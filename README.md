# Claude Code for an Academic Researcher

Made by a humble PhD student. A complete Claude Code infrastructure for researchers ŌĆö skills, agents, hooks, and rules for academic workflows. Built for researchers who write papers in LaTeX, manage bibliographies, run experiments, and want AI assistance that understands academic conventions.

Works on **macOS, Linux, and Windows**. Use Claude Code from the [terminal CLI](https://docs.anthropic.com/en/docs/claude-code), [VS Code](https://marketplace.visualstudio.com/items?itemName=anthropics.claude-code), [JetBrains IDEs](https://plugins.jetbrains.com/plugin/27189-claude-code), [the web](https://claude.ai/code), or the [desktop app](https://claude.ai/download) ŌĆö all share the same skills, agents, and rules.

## Installation

### Quick Install (npm)

```bash
npx flonat-research
```

This downloads the package and runs the setup script, which symlinks skills, agents, hooks, and rules into `~/.claude/`.

### Full Install (recommended for customisation)

#### macOS / Linux

```bash
git clone https://github.com/flonat/claude-research.git
cd claude-research
./scripts/setup.sh
```

#### Windows (PowerShell)

```powershell
git clone https://github.com/flonat/claude-research.git
cd claude-research
.\scripts\setup.ps1
```

The git clone gives you a local copy you can fully customise ŌĆö edit `.context/profile.md`, `CLAUDE.md`, and workflows to match your research.

### Update

```bash
# macOS/Linux: pull latest, then re-link without overwriting settings
git pull && ./scripts/setup.sh --update

# Windows (PowerShell):
git pull; .\scripts\setup.ps1 -Update
```

Then customise `.context/profile.md`, `.context/current-focus.md`, and `CLAUDE.md` with your details. See [`docs/getting-started.md`](docs/getting-started.md) for the full guide (includes Windows-specific setup, Python install, and troubleshooting).

### Your Fork on a New Machine

This repo is set up with two remotes: `origin` ŌåÆ your fork (`https://github.com/garrett-seo/claude-research.git`), and `upstream` ŌåÆ the original template (`https://github.com/flonat/claude-research.git`), so you can keep pulling template updates without losing your customisations.

On a new machine:

```bash
git clone https://github.com/garrett-seo/claude-research.git
cd claude-research
git remote add upstream https://github.com/flonat/claude-research.git
./scripts/setup.sh
```

Then recreate the local-only Python venv used by `hooks/pdf-to-markdown.py` (not tracked in git ŌĆö venvs aren't portable across machines):

```bash
uv venv hooks/venv
uv pip install -r hooks/requirements.txt --python hooks/venv/bin/python
```

To pull future updates from the template later:

```bash
git fetch upstream
git merge upstream/main   # or: git rebase upstream/main
git push origin main
```

> **Note:** `hooks/pdf-to-markdown.py` isn't currently wired into any hook event in `.claude/settings.json` ŌĆö register it under the relevant `PreToolUse` matcher if you want it active.

## Components

**Context Library** (`.context/`) ŌĆö Markdown files that give Claude persistent memory about you, your projects, and your workflows. Instead of re-explaining your research every session, Claude reads these files automatically.

**Skills** (`skills/`) ŌĆö Slash commands invoked with `/<skill-name>` or natural language.

50 skills available. Key examples: `/proofread`, `/literature`, `/bib-validate`, `/init-project-research`, `/pre-submission-report`, and more.
<!-- SKILLS-SUMMARY:END -->
See [`docs/skills.md`](docs/skills.md) for the full catalogue.

**Agents** (`.claude/agents/`) ŌĆö Specialised personas for complex review tasks, spawning sub-agents for parallel work.

<!-- AGENTS-TABLE:START -->
| Agent | Use case |
|-------|----------|
| `artifact-coherence-auditor` | Audits coherence between paper prose and replication outputs ŌĆö catches hallucinated results, missing scripts, mismatched numbers, and unverifiable claims |
| `blindspot` | Peripheral vision audit for empirical output |
| `claim-verify` | Verify that cited claims in a paper accurately represent what the source papers actually say |
| `code-paper-auditor` | Use this agent when you need to verify code-paper consistency ŌĆö mapping every quantitative claim in a paper to its source code and output files |
| `code-review` | Multi-persona orchestrator for adversarial review of R, Python, Julia, or Stata research scripts |
| `codex-research` | Code review and research agent that delegates to OpenAI Codex CLI in headless mode |
| `domain-reviewer` | Research-focused substantive correctness agent |
| `fatal-error-check` | Fast pre-review check for fatal errors in LaTeX papers |
| `fixer` | Generic fix implementer for any critic report |
| `gemini-research` | Web research agent that delegates to Gemini CLI in headless mode |
| `paper-critic` | Adversarial auditor for LaTeX papers |
| `peer-reviewer` | Use this agent when you need to review someone else's paper ŌĆö as a peer reviewer, discussant, or for reading group preparation |
| `proposal-reviewer` | Use this agent when you need to review a research proposal, extended abstract, conference submission outline, or pre-paper plan ŌĆö either his own or someone else's |
| `referee2-reviewer` | Use this agent when the user wants a rigorous, adversarial academic review of their work ŌĆö including papers, manuscripts, research designs, code, or arguments |
| `reproducibility-auditor` | Reviews research workflows for reproducibility gaps ŌĆö hidden dependencies, absolute paths, undocumented prerequisites, environment assumptions, and output traceability |

**Hooks** (`hooks/`) ŌĆö Automated guardrails that run at specific points in a session.

**Rules** (`.claude/rules/`) ŌĆö Always-on policies enforcing good research practices. See [`docs/rules.md`](docs/rules.md).

**Research Vault** ŌĆö Obsidian-style markdown vault (`~/Research-Vault`) for tasks, pipeline, submissions, venues, people, and themes. Accessed via the `taskflow` MCP server.

**Biblio MCP** ŌĆö Multi-source scholarly search server (OpenAlex + optional Scopus & Web of Science). See [`docs/bibliography-setup.md`](docs/bibliography-setup.md).

**Flonat-Papers MCP** ŌĆö Zotero library management server (search, PDF extraction, semantic retrieval, BibTeX export). Lives in `packages/flonat-papers/` with bundled `bib-validate` and `bib-parse` skills.

**Council Mode** ŌĆö Multi-model deliberation with 3 LLM providers, anonymised cross-review, and chairman synthesis. See [`docs/council-mode.md`](docs/council-mode.md).

## Session Continuity

<!-- SESSION-CONTINUITY:START -->
<!-- synced from private README ŌĆö do not edit manually -->
Each session builds on previous ones:

- `current-focus.md` ŌĆö updated at session end with progress and next steps
- `log/` ŌĆö timestamped session logs
- `log/plans/` ŌĆö saved implementation plans
- `MEMORY.md` ŌĆö accumulated `[LEARN]` tags (notation, citation, code, method, domain corrections)

The recovery protocol reads the latest plan, session log, and current focus to resume seamlessly.

## Design Principles

<!-- DESIGN-PRINCIPLES:START -->
<!-- synced from private README ŌĆö do not edit manually -->
1. **Lazy prompting** ŌĆö Context files eliminate repetitive explanations
2. **Hybrid local + cloud** ŌĆö Markdown (versioned) + Research Vault (dynamic)
3. **Question-driven** ŌĆö AI asks questions before dumping lists
4. **Read-only audits** ŌĆö Proofread, validate, review ŌĆö never auto-edit source
5. **Session continuity** ŌĆö Every session makes the next one better
6. **Permission governance** ŌĆö Global settings propagate automatically
<!-- DESIGN-PRINCIPLES:END -->
